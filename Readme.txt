1.Задача приложение - поиск изменившихся тегов в структуре xml файлов. 2 файла приложены к проекту
2.Изменения тегов происходит путем сравнения хэша текущей версии с исторической версией в классе SimpleExtendedReportSaver
3.Есть несколько таблиц второго уровня, которые хранят в себе ссылку на родителя например FinPeriod и Finstr
4.Сейчас механизм сравнения хэшей отключен, так как я провожу тестирование насквозь
5.Класс Inserter распараллеливает каждый поступивший sparkid в отдельную ветку вичислений
6.Класс SimpleExtendedReportSaver подсчитывает количество изменившихся тегов и запрашивает класс контейнер ExtendedReport
  для создания новых сущностей в БД
7.ExtendedReport создает объект CommonTable, который в себе инкапсулирует механизм сохранения всех новых сущностей
  и вызова процедур сброса метки curr
8.Все сущности описаны в пакете Models
9.Новые сущности создают креаторы, описанные в пакете Creators. 1 сущность - 1 креатор
10.Механизм историчности организован через метки curr=1 в БД, сущность без метки - история, curr = 1 актуальная
11.Сброс меток происходит через вызов хранимых процедур, заранее созданных в БД.
